# iFlow2API EXE 前端 UI 设计方案

## 参考风格分析

参考图片展示了一个现代化的深色主题管理面板：
- **配色**: 深蓝/深灰背景 (#1a1d29, #252a3c)，紫色/蓝色强调色
- **布局**: 顶部导航栏 + 主内容区卡片式布局
- **组件**: 开关按钮、下拉菜单、可折叠面板、图标按钮
- **风格**: 简洁现代，圆角设计，微妙阴影

## 功能映射

根据项目现有功能，设计以下页面：

### 1. 仪表盘 (Dashboard)
- 服务状态卡片（运行中/已停止）
- 实时统计：总请求数、成功率、活跃连接
- 系统资源：CPU/内存使用率
- 快捷操作按钮

### 2. 账号管理 (Account)
- OAuth 认证状态
- API Key 配置
- 凭证管理（查看/刷新/登出）
- 认证按钮（启动 OAuth 流程）

### 3. API 反代 (Proxy)
- 服务开关（启动/停止）
- 端口配置（默认 8000）
- 上游 API 地址配置
- 高级设置：重试次数、超时时间

### 4. 流量日志 (Logs)
- 实时请求日志列表
- 筛选功能（按模型、状态码、路径）
- 详情展开（Headers、Body、Thinking）
- 清空日志按钮

### 5. Token 统计 (Statistics)
- 请求统计图表
- 成功率趋势
- 模型使用分布
- 错误分析

### 6. 模型路由 (Model Router)
- 可用模型列表
- 模型别名映射
- 思考模式配置
- 默认模型设置

### 7. 设置 (Settings)
- 主题切换（暗黑/明亮）
- 语言选择
- 日志级别
- 缓存管理
- 关于信息

## 技术选型建议

### 方案 A: PyQt5 + QSS（推荐，与现有代码兼容）
- 优点：与现有 gui_pyqt.py 兼容，单文件可执行
- 缺点：样式定制相对有限
- 工作量：中等

### 方案 B: Tauri + React（现代化）
- 优点：现代化的 Web UI，美观度高
- 缺点：需要引入 Rust + Node.js 工具链
- 工作量：较大

### 方案 C: Flet (Flutter-based)
- 优点：Python 原生，现代化组件
- 缺点：打包体积较大
- 工作量：中等

## 推荐：方案 A - PyQt5 现代化改造

保留 PyQt5 技术栈，但进行现代化 UI 改造：
1. 重新设计界面布局（参考 Antigravity Tools 风格）
2. 使用 QSS 定义深色主题
3. 添加页面导航（侧边栏或顶部标签）
4. 美化组件样式（卡片、开关、按钮）
